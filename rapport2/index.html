Projekt 2, Rapport


1. Memory

Överlägset den mest frustrerande uppgiften. Strulade i 5h+ med att få kortet att svänga sig fram och tillbaka. Jag lyckades redan ganska snabbt få kortet att svänga sig men hiffade inte hur jag skulle "nolla" kortet till dess ursprungs ställning, turns out att det gick att göra med 3 rader kod:

flip-card .flip-card-inner.flip-card { transform: rotateY(180deg); }

Efter jag implementerade koden var det smooth sailing. Sist och slutligen inte en så svårt uppgift, relativt lärorik, och väldigt frustrerande.

2. Bildgalleri

Bildgalleriet var relativt svårt eftersom jag inte fick det och fungera med JS utan istället gjorde faden med css keyframes. Jag strulade lite och det ända sättet jag fick hover att fungera var om jag gjorde en container för hela galleriet. Vilket var väldigt konstigt eftersom min logik sa att det skulle borda fungera med .thumb:hover #gallery... men det fungerade inte?? Sist och slutligen lyckades jag någotlunda med uppgiften så länge man går ut ur containern.

3. Byt tema med HTML5 Sliders

Överlägset den mest tidskrävande uppgiften dock skulle jag inte säga att det var den svåraste. Uppgiften bestod till största delen av variabler eftersom man måste hantera r,g och b skillt. Denhär uppgiften var som sagt inte så svår men krävde mycke tid och jag måste flera gånger på google föratt läsa om teori både för hexa, octal och binär men mest för komplementär färgena. Dock när man väl fick färgena var det lätt att ändra färgen på ett element på sidan. En av de mest lärorika uppgifterna eftersom hexadecimal, binär och octal ännu var oklara. Nu förstår jag bättre teorin bakom dem och kan därmed hantera dom bättre och effektivare i framtiden.

4. DOM manipulering

Då vi väl började använda jQuery och man började fatta strukturen och implementations möiligheterna, förstod man hur otroligt bra verktyg jQuery är. Jag använde jQuery bl.a. för att fadea, visa och gömma, ändra storleken på element, för att färga rubriker och det var ändast stilistiska ändringar, jag använde jQuery genom projketet för mycke annat också! Väldigt lärorik uppgift och väldigt bra att man kan nu använda ett så starkt verktyg.

5. Användarhantering

Definitivt en av de svårare uppgifterna. Det kändes som att jag inte förstod riktigt vad cookies innebär och hur man kan editera och manipulera dom. Dock lyckades jag visa en hemlis då man loggar in med zinned. Svår uppgift och känns som att jag behöver mera info om cookies!

6. AJAX

Då man väl hade koden för att hämta in ett element var det lätt att göra samma sak till många olika element eller .txt filer. Jag började genast med att sätta in bilder på mina .txt filer och hämta in dom. jQuery versionen av uppgiften var ännu lättare när man väl börjat fatta hur jQuery fungerar. Denna version var lätt att hantera och med hjälp av den hämtade jag in en till bild och lite text samt en länk till wikipedia.

7. JSON

Eftersom vi gick egenom uppgiften under timmen var det lätt att sätta sig in i teorin bakom koden. Svårast var att förstå hur man skulle få data från openweathermap. Det bestod av trial and error men sist och slutligen hiffade jag mig på hur ja skulle få datan. Annors kändes uppgiten relativt lätt. OBS! För att se ip-api info måste URL vara oskyddad (http://), och inte skyddad (https://).

8. Charts

Uppgift 8 bestod till största delen av joinksande men dock var HighCharts och ChartJS något lunda struliga. GoogleCharts var lätta att förstå och det gick relativt lätt att ändra charten till sådan som jag ville ha den. Dock strulade jag mer med HighCharts och ChartJS eftersom det var svårare att hitta information om dem

Feedback

Summerat tycker jag att detta projekt gav väldigt mycket info om interaktivitet och gjorde det på ett intressant sätt. Personligen tycker jag att de mest lärorika uppgifterna var 3,4,6,7. Överlag kändes det som att projekt 2 gick smidigare än projekt 1 eftersom man visste bättre vad som krävdes för att få projektet färdigt. Samtidigt var man mycket bättre på JS vid projekt 2 än vid projekt 1 och det såklart ökade mycket på smidigheten. Jag tycker att ändringen till specifikations dokumentet var väldigt bra och ju mer specifikt dokumentet är destu mindre stressar man då man kan göra ett upplägg på vad allt som måst göras.

Sammanlagt: 37h 53 min
Föreläsningar: 13h 15 min (OBS jobbade såklart på projektet under denhär tiden)
Självstudier: 24h 31 min
